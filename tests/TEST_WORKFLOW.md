# MeetScribe 全功能测试工作流程

> 目标: 最少步骤测试所有功能，验证已知 bug 已修复

## 测试前准备

1. 确保音频文件存在: `recordings/` 目录下有测试音频
2. 确保模型已下载: `models_cache/models/` 目录存在

## 测试步骤

### 步骤 1: 启动应用

```bash
cd C:\MeetScribe
python src/main.py
```

**预期**: GUI 正常启动，无报错

### 步骤 2: 测试线上会议录制 (dual 模式)

1. 在录音模式下拉菜单选择 **"线上会议"**
2. 打开电脑媒体播放器，播放一段音频
3. 点击 **"开始录音"**
4. 等待 10 秒
5. 点击 **"停止录音"**

**预期**:
- 录音正常启动，无 SystemError (ERR-009)
- 系统音频流正常打开，无 Invalid device (ERR-010)
- 录音文件保存成功

### 步骤 3: 测试转写功能

1. 在文件列表中选择刚录制的文件
2. 点击 **"开始转写"**
3. 等待转写完成

**预期**:
- 转写正常完成
- 转写结果包含麦克风和系统音频的内容

### 步骤 4: 测试文件操作

1. 点击 **"打开结果"** 按钮
2. 点击 **"预览"** 按钮
3. 点击 **"发言人管理"** 按钮

**预期**:
- 文件操作正常，无报错 (ERR-011)
- 弹窗正常显示

### 步骤 5: 测试设置页

1. 点击侧边栏 **"设置"**
2. 修改一个设置项
3. 点击 **"保存"**
4. 返回主页

**预期**:
- 设置保存成功
- 返回主页正常

### 步骤 6: 测试现场会议录制 (mic 模式)

1. 在录音模式下拉菜单选择 **"现场会议"**
2. 点击 **"开始录音"**
3. 说几句话
4. 点击 **"停止录音"**
5. 进行转写

**预期**:
- 录音和转写正常
