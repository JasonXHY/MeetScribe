## 2. 声纹匹配失败（P0）

### 2.1 问题描述

之前匹配成功的音频文件（如 `26061116会议.wav`）现在匹配失败，日志显示 embedding 维度问题。

### 2.2 旧版对比位置

| 对比项 | 旧版文件 | 新版文件 |
|--------|----------|----------|
| 声纹匹配 | `src_old/transcription.py:594-684` | `C:\侧耳倾听\src\gui\transcription.py:324-389` |
| 声纹库 | `src_old/voiceprint.py`（无） | `C:\侧耳倾听\src\voiceprint.py` |

### 2.3 根因分析

**问题链路**：
1. CAM++ `inference()` 返回 `[2, 192]` 或 `[1, 1, 192]` 的嵌入向量
2. 新版代码只剥一层维度，导致 embedding 形状不一致
3. `_cosine_similarity()` 维度校验失败，返回 0.0

**旧版处理**：
```python

# 旧版直接传入，未做维度处理
```

**新版问题**：
```python
# transcription.py:349-350
emb_array = np.array(embedding) if not isinstance(embedding, np.ndarray) else embedding
logger.info(f"[VOICEPRINT] Speaker {speaker_id + 1}: embedding dim={emb_array.shape}")
# 日志测量的是元组长度(2)，非 embedding 维度
```

**根本原因**：
1. `_send_embeddings` 中 `len(emb)` 测量的是 `(embedding_list, quality)` 元组长度（2），不是 embedding 维度
2. 需要取 `emb[0]` 测量实际 embedding 维度
3. `np.squeeze()` 应用于 embedding 而非元组

### 2.4 修复方案

**修复点 1**：`transcription.py:349` 修正维度测量

```python

# 修复后

    emb_array = np.array(embedding[0])
else:
    emb_array = np.array(embedding) if not isinstance(embedding, np.ndarray) else embedding
emb_array = np.squeeze(emb_array)  # 自动去掉所有 size=1 的维度
```

**修复点 2**：`voiceprint.py:358-366` 增强维度校验

```python

    # ... 后续计算
```

## 4. 音色库评分机制未生效（D）

### 4.1 问题描述

音色库中的质量评分（quality）未正确影响匹配结果，高评分样本未获得更高权重。

### 4.2 旧版对比位置

| 对比项 | 旧版文件 | 新版文件 |
|--------|----------|----------|
| 评分使用 | `src_old/transcription.py:668` | `C:\侧耳倾听\src\gui\transcription.py:433` |
| 声纹库 | `src_old/voiceprint.py`（无） | `C:\侧耳倾听\src\voiceprint.py:99-105` |

### 4.3 根因分析

**新版评分机制**：
```python
# voiceprint.py:99-105
def get_weighted_embedding(self):
    """获取加权平均声纹向量（高质量样本权重更大）"""
    if not self.embeddings:
        return None
    vectors = [e["vector"] for e in self.embeddings]
    weights = [e["quality"] for e in self.embeddings]
    return np.average(vectors, weights=weights, axis=0)
```

**问题点**：
1. `get_weighted_embedding()` 使用 `np.average()` 加权平均，**但 quality 值可能为 0 或 None**
2. 当 quality=0 时，该样本权重为 0，完全不参与平均
3. 新版 `_apply_voiceprint_match()` 中 `quality = self._speaker_qualities.get(speaker_id, DEFAULT_SPK_QUALITY)` 可能返回 0

**旧版处理**：
```python
# src_old/transcription.py:668
quality = self._speaker_qualities.get(speaker_id, 0.85)
# 旧版默认值 0.85，确保所有样本都有正权重
```

### 4.4 修复方案

**修复点 1**：`voiceprint.py:103` 确保 quality > 0

```python
def get_weighted_embedding(self):
    """获取加权平均声纹向量（高质量样本权重更大）"""
    if not self.embeddings:
        return None
    vectors = [e["vector"] for e in self.embeddings]
    # 确保 quality > 0，避免零权重样本
    weights = [max(e.get("quality", 0.85), 0.01) for e in self.embeddings]
    return np.average(vectors, weights=weights, axis=0)
```

**修复点 2**：`transcription.py:433` 确保默认 quality 有效

```python
# 修复前
quality = self._speaker_qualities.get(speaker_id, DEFAULT_SPK_QUALITY)

# 修复后
quality = self._speaker_qualities.get(speaker_id, DEFAULT_SPK_QUALITY)
if quality is None or quality <= 0:
    quality = DEFAULT_SPK_QUALITY  # 确保有效值
```

### 4.5 验证要点

- [ ] `get_weighted_embedding()` 返回有效向量
- [ ] 高 quality 样本对匹配结果有更大影响
- [ ] quality=0 的样本不会导致匹配失败

> **[QW-v2] 复核意见**：
>
> **问题严重程度降低**：核实 `DEFAULT_SPK_QUALITY = 0.85`（styles.py:57），不是 v1 猜测的 0。而且 `_process_message()` 中拆分 tuple 后，`_speaker_qualities` 存的是 transcriber 计算出的实际 quality（基于 segment count + duration），正常值范围 [0.5, 1.0]。
>
> **修复点 1 仍值得做**：`get_weighted_embedding()` 中加 `max(quality, 0.01)` 是合理的防御——如果音色库 JSON 中存了 quality=0 的历史脏数据（dim=2 bug 写入的），当前代码确实会导致该样本权重为 0。
>
> **修复点 2 不需要**：`_apply_voiceprint_match()` line 433 的 `self._speaker_qualities.get(speaker_id, DEFAULT_SPK_QUALITY)` 已经有 0.85 兜底，不会是 0。

### 7.1 问题描述

~~转写完成后，文件列表的"主题"列为空。~~ **已修复，无需处理。**

### 7.2 根因分析

`_extract_topic_from_summary()` (transcription.py:642-662) 的逻辑错误——把 `### 会议主题` 这个 section header 当 topic 文本提取了。

实际 test2_summary.md：
```
### 会议主题
干部管理系统一期使用反馈与二期优化规划讨论
```

代码跟踪：第 1 行 `### 会议主题` → `lstrip("# ")` 得到 `"会议主题"`（4 字符）→ `len > 5` 为 False → 跳过。第 2 行正文不以 `#` 开头 → 不匹配。**函数返回 `None`**，`update_topic()` 从未被调用。

### 7.3 修复方案

跳过 header 行，取下一行正文内容作为 topic：
```python
def _extract_topic_from_summary(self, summary):
    try:
        lines = summary.splitlines()
        for i, line in enumerate(lines[:15]):
            line = line.strip()
            if line.startswith("#") and ("主题" in line or "会议" in line):
                for j in range(i + 1, min(i + 4, len(lines))):
                    next_line = lines[j].strip()
                    if next_line and not next_line.startswith("#"):
                        topic = next_line[:50]
                        if len(topic) > 2:
                            return topic
                break
            if "主题" in line and "：" in line:
                m = re.search(r'主题[：:]\s*(.+)', line)
                if m:
                    return m.group(1).strip()[:50]
    except Exception:
        pass
    return None
```

## 8. Queue poll error（P2）

> **[QW-v2] 新增问题**：v1 文档未收录，来自 `docs/v1.0-frontend-issues.md` 排查。

### 8.1 问题描述

`'NoneType' object has no attribute 'get_nowait'`

### 8.2 根因分析

`_on_done()` 中先 `self._poll_timer.stop()` 再 `self._queue = None`。但 Qt timer event 可能在 `stop()` 前已入队，在 QMessageBox 嵌套事件循环中被 dispatch，此时 `self._queue` 已为 None。

### 8.3 修复方案

在 `_poll()` 的 while 循环内加防御性 None 检查：
```python
while True:
    try:
        if self._queue is None:
            break
        msg = self._queue.get_nowait()
        self._process_message(msg)
    except queue.Empty:
        break
```
