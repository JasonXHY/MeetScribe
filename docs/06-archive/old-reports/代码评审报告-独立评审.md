> 评审依据：`requirements.md` v3.0 + `architecture.md` v3.0 + `ui-redesign-guide.md` v1.1 + HTML 原型图 + 全部核心代码

## 总评

侧耳倾听项目整体架构清晰，分层合理（GUI → Bridge → Core → Engine），多进程转写设计避免了 GIL 限制，Lucide 图标系统和 QTableWidget 文件列表的引入显著提升了 UI 一致性。代码风格统一，日志体系完善。但存在若干 P0 级功能缺陷（声纹保存变量名错误、AI 纠错默认值类型不一致、多进程设备参数硬编码），以及架构层面的配置管理、线程安全等问题需要关注。

## P0 — 严重问题（影响功能/导致崩溃）

### P0-1: AddVoiceDialog 保存声纹使用错误变量名

**文件**: `src/gui/voiceprint_page.py:282,290`

**问题描述**: `_save()` 方法中引用了 `self._embedding`，但声纹数据实际存储在 `self._temp_embedding`（在 `_on_audio_process_done` 中赋值，第 256 行）。`self._embedding` 从未被赋值，始终为 `None`。

**影响范围**: 用户在音色库页面录音添加音色时，保存操作会直接触发 `QMessageBox.warning` 提示"请先录音提取声纹"，即使声纹已成功提取。

**建议修复**:
```python
# 第 282 行
if self._temp_embedding is None:
# 第 290 行
library.add_speaker(name, self._temp_embedding, "manual_recording", quality=0.90)
```

### P0-2: 多进程转写设备参数硬编码为 "cpu"

**文件**: `src/gui/transcription.py:143`

**问题描述**: `_execute_task()` 创建子进程时，`device` 参数硬编码为 `"cpu"`（第 143 行），完全忽略了用户在设置页选择的运算设备（`_device_var` / 配置 `device`）。

```python
self._process = multiprocessing.Process(
    target=transcribe_worker_process,
    args=(
        self._queue,
        MODEL_CACHE_DIR,
        "cpu",  # ← 硬编码，应从 config 读取
        ...
    ),
)
```

**影响范围**: 用户在设置页选择 "CUDA (GPU)" 后，转写仍然使用 CPU，GPU 加速完全失效。

**建议修复**:
```python
device = self._app.config.get("device", "CPU") if self._app else "cpu"
device = "cuda" if "cuda" in device.lower() else "cpu"
```

### P0-3: AI 纠错/摘要配置值类型不一致导致功能静默失效

**文件**: `src/gui/transcription.py:221-240`, `src/config.py:22`

**问题描述**: `config.py` 默认值中 `auto_correction = False`（布尔值），但 `settings_page.py` 保存时写入字符串 `"转写后自动纠错"` 或 `"关闭"`。`transcription.py` 第 221-227 行的判断逻辑虽然做了兼容处理（检查布尔值和字符串两种格式），但 `_get_ai_service()` 在 `auto_correction` 为 `False` 时仍然会尝试获取 AI 服务，导致不必要的初始化开销。更关键的是，**首次使用时默认配置不触发 AI 摘要**（`auto_summary` 在默认配置中不存在），需要用户手动去设置页开启。

**影响范围**: 首次使用用户不知道需要手动配置 AI 摘要，核心差异化功能（自动会议纪要）在出厂默认配置下不触发。

**建议修复**: 在 `config.py` 的 `DEFAULTS` 中将 `auto_correction` 默认值改为 `"转写后自动纠错"`，`auto_summary` 默认值改为 `"转写后自动生成"`。

### P0-4: 声纹匹配后更新结果文件时 speaker_id 类型不一致

**文件**: `src/gui/transcription.py:347`

**问题描述**: `_match_voiceprints()` 中调用 `apply_speaker_mapping` 时传入 `{speaker_id + 1: name}`（int 作为 key），但 `apply_speaker_mapping` 在 `utils.py` 中期望的 key 类型可能是字符串（与 `item.speaker_names` 的格式一致：`{"0": "张三"}`）。这可能导致声纹匹配结果无法正确写入结果文件。

**影响范围**: 声纹自动匹配功能（F-1 问题）虽然在代码中已实现，但因类型不一致可能导致匹配结果未正确持久化到结果文件。

**建议修复**: 确保 `apply_speaker_mapping` 内部对 key 做 `str()` 转换，或在调用时统一使用字符串 key。

## P1 — 中等问题（影响体验/存在隐患）

### P1-1: 默认配置路径硬编码，不适用于其他用户

**文件**: `src/config.py:16-17`

**问题描述**: `DEFAULTS` 中录音和转写目录硬编码为 `C:\MeetScribe\recordings` 和 `C:\MeetScribe\transcripts`，但实际项目位于 `C:\侧耳倾听\`。这导致默认路径不存在时配置中保存的是无意义路径。

**建议修复**: 使用相对于项目根目录的路径，或使用 `os.path.expanduser("~/MeetScribe")` 作为默认值。

---

### P1-2: 多进程资源清理异常被静默吞掉

**文件**: `src/gui/transcription.py:274-283`

**问题描述**: `_on_done()` 中清理子进程资源时使用了 bare `except Exception`，吞掉了所有异常。如果 `terminate()` 或 `join()` 超时，不会有任何日志记录。

```python
try:
    if self._process:
        if self._process.is_alive():
            self._process.terminate()
        self._process.join(timeout=2)
    self._process = None
    self._queue = None
except Exception:  # ← 吞掉所有异常
    self._process = None
    self._queue = None
```

**建议修复**: 至少添加 `logger.warning()` 记录异常信息，便于排查问题。

---

### P1-3: FileManager 后台线程获取时长存在竞态条件

**文件**: `src/file_manager.py:180-184,188-200`

**问题描述**: `_fetch_duration_async` 在 daemon 线程中修改 `item.duration_s`，然后调用 `_notify()` 和 `_save_to_file()`。虽然 CPython 的 GIL 保证了简单属性赋值的原子性，但 `_save_to_file()` 中的 JSON 序列化可能读到不一致的状态（例如 `duration_s` 刚被更新但 `file_size` 还没）。注释中提到了这个问题，但 `_save_to_file()` 使用的 `os.replace()` 只保证文件写入的原子性，不保证内存状态的一致性。

**建议修复**: 使用 `threading.Lock` 保护 `_files` 列表的读写操作，或在 `_fetch_duration_async` 中使用 `QTimer.singleShot(0, ...)` 回到主线程更新。

---

### P1-4: 通知脚本存在命令注入风险

**文件**: `src/gui/app.py:407-419`

**问题描述**: `_send_notification` 中通过 f-string 拼接 PowerShell 脚本，虽然做了单引号转义（`replace("'", "''")`），但 `safe_title` 和 `safe_message` 中如果包含 `"` 字符，可能导致 PowerShell 脚本语法错误。更安全的方式是通过 stdin 传递参数。

**建议修复**: 使用 `subprocess.Popen` 的 `stdin` 参数传递参数，避免拼接脚本。

---

### P1-5: home_page.py `_on_progress_updated` 属性访问不安全

**文件**: `src/gui/home_page.py:557-559`

**问题描述**: `_on_progress_updated` 直接访问 `progress.stage` 和 `progress.percent`，但 `progress` 对象可能不是 `TranscriptionProgress` 类型（信号传递时可能为其他类型），缺少 `hasattr` 保护。

**建议修复**: 添加类型检查或使用 `getattr(progress, 'stage', '')` 安全访问。

---

### P1-6: 文件列表空状态无提示信息

**文件**: `src/gui/file_list_view.py`

**问题描述**: 当文件列表为空时，`QTableWidget` 显示空白区域，没有提示用户"请添加音频文件"或类似引导信息。新用户可能不知道下一步该做什么。

**建议修复**: 在 `set_files([])` 时，添加一个居中的空状态提示标签。

---

### P1-7: SettingsPage 配置保存缺少输入验证

**文件**: `src/gui/settings_page.py:617-660`

**问题描述**: `_on_save()` 方法直接将 UI 控件的值写入配置，没有验证：
- 录音目录是否存在、是否可写
- API Key 格式是否正确
- 各 ComboBox 值是否在合法范围内

**建议修复**: 在保存前验证关键配置项（特别是路径和 API Key）。

---

### P1-8: VoiceprintPage AddVoiceDialog 使用 WindowStaysOnTopHint

**文件**: `src/gui/voiceprint_page.py:66`

**问题描述**: `AddVoiceDialog` 设置了 `Qt.WindowStaysOnTopHint`，这会导致弹窗始终置顶，可能遮挡其他窗口，影响用户体验。在 Windows 11 上，这个 flag 还可能导致弹窗无法正确获得焦点。

**建议修复**: 移除 `Qt.WindowStaysOnTopHint`，改用模态对话框（`setModal(True)`）。

---

### P1-9: unified_recorder.py 缺少录音超时机制

**文件**: `src/unified_recorder.py`

**问题描述**: 录音模块没有超时保护。如果音频设备挂起或 PyAudio 回调异常，录音线程可能永远阻塞，UI 状态无法恢复。

**建议修复**: 添加最大录音时长限制（如 4 小时），超时后自动停止录音。

---

### P1-10: 文件历史持久化在每次添加文件时触发写入

**文件**: `src/file_manager.py:175`

**问题描述**: `add_file()` 每次被调用时都会执行 `_save_to_file()`，在批量添加文件时（如拖拽多个文件）会导致频繁的磁盘 I/O。

**建议修复**: 使用防抖机制，或在批量操作结束后统一保存。

## P2 — 轻微问题（代码质量/可维护性）

**建议修复**: 要么删除这个字典，要么使用 `__getattr__` 结合类型提示实现真正的类型安全。

### P2-4: 文件大小格式化不一致

**文件**: `src/file_manager.py:107-111` vs `src/gui/home_page.py:860-869`

**问题描述**: `AudioFile.size_str` 使用 `1024 * 1024` 阈值显示 MB，但 `HomePage._format_size` 也使用相同阈值但格式不同（`{size_bytes / (1024 * 1024):.1f}MB` vs `{self.file_size / (1024 * 1024):.1f} MB`）。两处逻辑重复且格式不完全一致。

**建议修复**: 统一使用一个格式化函数，避免重复。

### P2-5: transcription.py summary_path 替换逻辑脆弱

**建议修复**: 逐步迁移所有引用到新常量系统，然后删除遗留常量。

### P2-7: settings_page.py API Key 容器冗余样式

**文件**: `src/gui/settings_page.py:216-218`

### P2-8: ai_service.py model 名称小写转换可能破坏厂商特定模型名

**文件**: `src/ai_service.py:72`

**问题描述**: `model_lower = self.model.lower()` 将模型名称转为小写用于 registry 查找。但如果某些厂商的模型名大小写敏感（如 "MiMo-V2.5-Pro"），转换后可能无法匹配。

### P2-9: voiceprint.py 去重阈值硬编码

**文件**: `src/voiceprint.py:69`

**问题描述**: 同源去重相似度阈值 `0.999` 直接硬编码在代码中，没有提取为常量。

**建议修复**: 提取为 `DEDUP_THRESHOLD = 0.999` 常量，与其他阈值（`MATCH_THRESHOLD`、`HIGH_CONFIDENCE`）保持一致。

### P2-10: first_launch.py 模型下载为模拟实现

**文件**: `src/gui/first_launch.py:38-49`

**问题描述**: `ModelDownloadWorker.run()` 中的模型下载是模拟实现（`time.sleep`），注释说"实际应从云端下载"。这意味着首次启动引导中的模型下载功能是无效的。

**建议修复**: 实现真实的模型下载逻辑，或在首次启动时跳过下载步骤，引导用户去设置页下载。

## 亮点

1. **架构分层清晰**: GUI → Bridge（TranscriptionHandler）→ Core → Engine 四层分离，职责明确，扩展性好。

2. **多进程转写设计**: 使用 `multiprocessing.Process` + `Queue` 实现子进程转写，避免 GIL 限制，崩溃隔离性好。

3. **Lucide 图标系统**: `gui/icons.py` 基于 SVG + QSvgRenderer 实现统一的线性图标，支持多 DPI，渲染一致。

4. **自适应轮询机制**: `TranscriptionHandler._poll()` 的轮询间隔自适应（50-500ms），平衡了响应速度和 CPU 占用。

5. **配置原子写入**: `Config.save()` 使用 `tempfile` + `os.replace()` 实现原子写入，防止写入中断导致配置损坏。

6. **文件变更防抖**: `MeetScribeApp._on_file_changed()` 使用 QTimer 防抖，避免频繁刷新文件列表。

7. **日志过滤系统**: `USER_FRIENDLY_KEYWORDS` 在三个入口（`_log`、`_append_log`、`_poll_queue`）中统一使用，只显示用户关心的信息。

8. **声纹匹配完整链路**: 从声纹提取 → 持久化 → 匹配 → 自动命名 → 结果更新，链路完整。

9. **测试覆盖**: 包含单元测试、集成测试、GUI 测试、端到端测试，覆盖了核心功能。

10. **UI 重新设计方案完整**: `ui-redesign-guide.md` 从色值、字体、间距、圆角到组件规范，体系化设计，HTML Mockup 可直接预览。
