### 🔴 F-2 — AI 摘要/纠错在出厂默认配置下永不触发（AI-001 / AI-002）

存在**两个内容不一致的 `settings.json`**，且**键名错配**：

## 3. 代码质量评审（Senior-Engineer 视角）

整体：**风格统一、注释充分、日志体系良好的合格原型，但类型安全、异常纪律、跨进程/配置契约存在系统性缺口**，距"生产可交付"仍有明确差距。

### 3.1 Agent-Readable（可读性）

**好的地方**：一致的中文 docstring、清晰命名、`══` 分区注释、无 `print` 残留、统一 `logging`。

**问题**：
1. **类型注解几近缺失：391 个 `def` 中仅 22 个有返回类型标注（约 5%）**。最严重的是跨进程队列协议——
   `_process_message(msg)` 按 `msg[0]` 分发、按 `msg[1..4]` 位置解包（`transcription.py:176-244`），对应
   `transcribe_worker.py` 中 11 处 `queue.put((...))`。要弄清 `msg[4]` 是 `transcript_path` 必须跨两个
   文件对照。一处元数错位 = 子进程边界静默 `IndexError`。
   - 改进：把位置元组升级为 `@dataclass`，用 `match/case` 分发，`case _` 兜底告警。
2. **字符串当枚举**（`auto_correction == "转写后自动纠错"`）：F-2 正是 stringly-typed 配置的直接后果。
3. **两个 `settings.json` + `src`/`src_old` 重复**：`diff` 证实后端文件**逐字节相同**，仓库体积近乎翻倍，
   Agent 需猜测哪棵树是活的。

### 3.2 Debuggable（可调试性）

**好的地方**：统一 `MeetScribe` logger、init 崩溃路径有 `traceback.format_exc()`、`app.__init__` 每步 debug 日志。

**问题**：
1. **`src/` 共 81 处 `except Exception`**（v1 在 GUI 子集统计为 42）。许多把真实堆栈降级成一行用户文案：
   `transcription.py:332,375` `except Exception as e: self.log_message.emit(f"...: {e}")`——**root cause 丢失**，
   无 `logger.exception()`。
2. **静默配置 miss**：F-2 中 `auto_summary` 缺失/不符时分支不触发，**无任何告警日志**——这正是 F-2 难以察觉的原因。
3. **`_safety_timer`（`app.py:226-249`）每 5 秒空跑**：只做 `hasattr` 检查 + 打日志，从不修复任何东西——
   安慰剂代码，徒增唤醒。建议删除或改为真正的子进程存活监控。

### 3.3 Robust（健壮性）

- **F-1/F-2** 本质是健壮性失效（功能丢失、配置死路径）。
- **子进程错误处理**：`_poll` 裸 `except Exception`（167 行）+ worker 整体 `try` 只发 `("error", str(e))`，
  用户只看到"转写失败: <str>"，真实堆栈最多落在一行日志里。
- **PowerShell 通知**（`app.py:403-417`）：输入为内部字符串，目前安全，但字符串拼接进 PS 脚本仍是脆弱模式；
  建议 `-EncodedCommand`（Base64）从根上消除注入面。
- **位置元组协议**（见 3.1）是跨进程边界的潜在崩溃面。
- **`.pyc` / `__pycache__/` 被提交进 git**（74 个），`.gitignore` 仅覆盖 zig 产物。

### 3.4 Testing（可测试性）

- **T-0（整套崩溃）**：`test_voiceprint_threshold.py:11` 在模块顶层 `sys.stdout = io.TextIOWrapper(...)`，
  关闭 pytest 捕获缓冲 → 任何整套运行都崩。**单行修复，杠杆最高。** 该文件实为手动诊断脚本，无测试函数。
- **测试被误标**：`test_transcription.py` / `test_file_list.py` 的失败被文档归为"旧 tkinter API"，实际抓到的是
  **真实回退（F-1、F-4）**。文档的失败归因是错的。
- `test_performance_optimization.py` 顶层 `import customtkinter`（死依赖）→ 整文件硬报错。
- 无 CI 配置、无默认 `QT_QPA_PLATFORM=offscreen`、`print("PASS")` + `if __name__=="__main__"` 风格混杂。
- 重复/重叠套件（`test_optimization` vs `test_performance_optimization`、`test_voiceprint` vs `test_voiceprint_fix`）。

### Fix 1 — 恢复声纹自动匹配（F-1，功能优先级最高）

```python
def _match_voiceprints(self) -> None:
    if not self._speaker_embeddings or self._voiceprint_matched:
        return
    self._voiceprint_matched = True
    from voiceprint import VoiceprintLibrary
    library = VoiceprintLibrary()
    for spk_id, emb in self._extract_speaker_embeddings().items():
        name, confidence = library.match_with_confidence(emb)   # 返回 (name, "confirmed"/"suggested"/"no_match")
        if not name:
            continue
        # ... 仅更新当前批次文件的 speaker_names + apply_speaker_mapping ...
        if confidence == "confirmed":
            library.add_speaker(name, emb, source=..., quality=...)
```
