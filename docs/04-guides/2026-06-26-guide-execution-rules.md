# 执行准则

> 创建日期：2026-06-26
> 目的：确保每次会话都能遵守规则，避免重复犯错

## 会话启动检查清单

每次会话开始必须执行：
- [ ] Read MEMORY.md 确认致命规则
- [ ] 确认工作目录是 `C:\侧耳倾听\`
- [ ] git pull 获取最新代码
- [ ] 检查是否有未完成的任务

## 修改文件前检查清单

每次修改文件前必须执行：
- [ ] 打印完整路径让用户确认
- [ ] git status 查看当前状态
- [ ] 小改动跑相关测试，大改动跑全量测试

## 打包前检查清单

每次打包前必须执行：
- [ ] 确认 PyInstaller 已安装（venv_build）
- [ ] 确认 ISCC 已安装
- [ ] 确认 me.spec 指向 `C:\侧耳倾听`
- [ ] 确认图标已更新
- [ ] 确认所有修改已测试通过
- [ ] 确认 utils.get_data_dir() 已在所有写入路径使用

## 文档管理规则

### 文档位置
- 所有文档统一放 `C:\侧耳倾听\docs\`
- 不要丢桌面

### 文档命名规范
格式：`[日期]-[类型]-[简述].md`

类型代码：
- `spec` = 需求规格
- `plan` = 实施方案
- `report` = 复盘报告
- `guide` = 开发指南
- `issue` = 问题记录
- `fix` = 修复记录

示例：
- `2026-06-26-plan-memory-restructure.md`
- `2026-06-26-report-packaging-review.md`
- `2026-06-26-issue-permission-error.md`

### 文档更新规则
更新前：
1. 读取当前文档确认结构
2. 确认更新内容属于哪个部分

更新时：
1. 保持文档结构清晰
2. 添加更新时间戳

更新后：
1. 验证文档完整性
2. 检查索引是否需要更新

## 记忆文件管理规则

### MEMORY.md 主文件规范
- 容量限制：必须控制在 40 行以内
- 必须包含：致命规则、项目文件结构、子文件索引表
- 禁止包含：详细历史记录、完整架构决策、具体修复细节

### 子文件目录结构
```
memory/projects/<project-id>/
├── MEMORY.md              # Trunk：主文件（<40行）
├── domains/               # 领域分支
│   ├── rules/             # 规则
│   ├── arch/              # 架构
│   ├── progress/          # 进度
│   ├── issues/            # 问题
│   └── tools/             # 工具
```

### 更新记忆文件的规则
更新前：
1. 读取当前 MEMORY.md 确认结构
2. 确认更新内容属于哪个 domain
3. 检查对应 _index.md 是否需要更新

更新时：
1. 主文件 MEMORY.md 只更新索引表和致命规则
2. 详细内容写入对应的 leaf 文件
3. 更新对应 _index.md 的摘要和 token 估算

更新后：
1. 验证 MEMORY.md 仍在 40 行以内
2. 验证索引表与 leaf 文件一致
3. 在 _index.md 中添加更新时间戳

## 禁止事项

1. 禁止修改 `C:\MeetScribe\`
2. 禁止凭猜测搜索外网（无 VPN）
3. 禁止在未确认的情况下打包
4. 禁止边改边打包
5. 禁止在未打印路径的情况下修改文件
