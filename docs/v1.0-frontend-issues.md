### 问题 4：声纹匹配写入逻辑（P1）

**现象**：多个 Speaker 匹配同一音色库成员时，最高分的误匹配仍被写入，导致 profile 污染。

**日志**：
```
[VOICEPRINT] Skipped conflict: Speaker 3 -> 万斌 (score=0.722, lower than best match)
```

**代码位置**：`src/gui/transcription.py` 第 317-369 行 `_match_voiceprints`

**根因**：冲突检测只对每个音色库成员保留 score 最高的 Speaker，但不检查这个最高分是否真的是正确匹配。

**排查方向**：
1. 是否应该在多 Speaker 匹配同一人时不写入声纹？
2. 是否应该添加最低分数阈值（如 0.85）？
3. 声纹写入前是否应该检查 profile 是否已被污染？
