## P1：API Key 提示文字不显示

### 现象
删除设置中的 API Key 后，提示文字（"使用内置 Key"等）未显示。

### 根因分析
`_api_key_hint` 标签在 `_build_ai_section()` 中创建时根据初始状态设置文字，但用户修改 API Key 后未动态更新提示文字。

<!-- [QW] 根因分析方向正确。修复方案可行，建议在 `_on_save()` 成功后调用一个 `_update_api_key_hint()` 方法，根据三种状态（用户自定义 Key / 使用默认 Key / 无 Key）分别显示不同提示文字和颜色。 -->

### 修复方案
1. 在 `_on_save()` 或 API Key 输入框的 `textChanged` 信号中，动态更新 `_api_key_hint` 文字和颜色
2. 逻辑：读取 `ai_user_api_key` 和 `ai_default_api_key`，根据状态更新提示

2. `file_manager.update_topic()` 是否被调用
3. `file_status_changed` 信号是否触发了列表刷新
4. 主题写入逻辑可能在 multiprocessing spawn 中被阻断

## P7：日志/状态逻辑问题

### 现象
1. 转写完成后按钮仍显示"转写中"，直到 AI 摘要结束才变为"完成"
2. 日志中转写和总结混在一起，未区分
3. 很多状态更新不到位

### 根因分析
`transcription.py` 的状态管理：
- 转写完成（`file_done`）时未立即更新 UI 状态
- AI 摘要（`auto_summary`）是异步的，但 UI 状态依赖摘要完成
- 日志消息未区分"转写完成"和"摘要生成中"

### 修复方案
1. **状态分离**：转写完成时立即更新状态为"已完成（待摘要）"，摘要完成后更新为"已完成"
2. **按钮更新**：转写完成后立即显示"预览/打开"等按钮，不等摘要
3. **日志分离**：明确区分"[转写完成]"和"[AI摘要]"日志前缀
4. **进度反馈**：转写和摘要分别显示进度

## P9：音色库评分机制未生效

### 现象
会议中的录音评分全是 1，评分机制未生效。

### 根因分析
需要排查：
1. `voiceprint.py` 中 `add_speaker()` 是否正确计算 quality
2. `transcription.py` 中 `_auto_add_speaker()` 是否传递了正确的 quality 参数
3. `DEFAULT_SPK_QUALITY` 的值是否正确

<!-- [QW] 代码验证结果——管道本身是完整的，但 quality 值来源是问题：
- `DEFAULT_SPK_QUALITY = 0.85`（styles.py 第 57 行）
- `add_speaker(name, embedding, source, quality=0.85)` 正确传递并存储 quality
- `_auto_add_speaker()` 使用 `self._speaker_qualities.get(speaker_id, DEFAULT_SPK_QUALITY)` 获取 quality
- quality 用于加权平均计算（`get_weighted_embedding()`），不是作为"评分"显示

**问题在于**：voiceprint_page.py 详情面板显示的"平均质量"值（如截图中的 0.85）来自 `profile.embeddings` 中每个 embedding 的 quality 字段。如果所有 embedding 都是默认 0.85，显示的就是 0.85。
用户说"评分全是 1"——如果用户看到的是 0.85 而不是 1，可能是显示精度问题。
**但核心问题是**：quality 没有实际评分逻辑。`_speaker_qualities` 字典的值来源于转写 worker 的 `spk_embeddings` 消息，如果 worker 不计算真实质量（例如 SNR、置信度等），所有样本都会得到默认值 0.85。
**建议**：(1) 确认 `_speaker_qualities` 是否在 worker 中被正确填充真实评分；(2) 如果 worker 未提供 quality，考虑基于 embedding 向量的范数或与平均向量的余弦相似度计算真实质量。 -->

### 修复方案
1. 检查 `_speaker_qualities` 字典是否在转写过程中正确填充
2. 检查 `add_speaker()` 的 quality 参数是否被正确使用
3. 检查 quality 评分逻辑是否实现（可能只是占位符）
