```python

    background-color: {C_ACCENT_LT}; color: {C_TXT1};

    background-color: #F9FAFB;
}
```

## 问题 B：发言人姓名回写不完整

**现象**：AI 总结文件中能看到正确匹配的姓名，但转写文件和发言人管理对话框中没有。

**根因分析**：

三套姓名回写机制按此顺序执行：
1. `_generate_summary()` 中的摘要提取回写（基于 AI 摘要）
2. `_apply_voiceprint_match()` 中的声纹匹配回写（基于 CAM++ 余弦相似度）
3. `_apply_speaker_names()` 中的正则/LLM 回写（基于转写文本）

**关键发现**：后一步会覆盖前一步的结果。如果声纹匹配的 confidence 不是 "confirmed"（score < 0.50），姓名会被后续步骤覆盖。

**可能的原因**：
- 声纹匹配的 score 低于阈值，姓名未被保留
- 或者写入操作时 `item.status != FileStatus.DONE`，跳过了写入
- 或者 `item.file_path not in self._current_batch_paths`，跳过了写入

**修复方案**：

文件：`C:\侧耳倾听\src\gui\transcription.py`

调试步骤（先添加日志，确认问题）：

在 `_apply_voiceprint_match()` 方法开头添加：
```python
logger.debug(f"[VOICEPRINT] Applying match: speaker_id={speaker_id}, name={name}, confidence={confidence}")
logger.debug(f"[VOICEPRINT] Current batch paths: {self._current_batch_paths}")
```

在写入文件前添加：
```python
logger.debug(f"[VOICEPRINT] Item status: {item.status}, result_path: {item.result_path}")
logger.debug(f"[VOICEPRINT] File path in batch: {item.file_path in self._current_batch_paths}")
```

根据日志结果修复：
- 如果 `item.status != FileStatus.DONE`：检查状态更新时机
- 如果 `item.file_path not in self._current_batch_paths`：检查 `_current_batch_paths` 的设置

## 问题 C：主题未回写到主页文件列表

**现象**：转写后主题列为空。

**根因分析**：

链路理论上完整：
```
update_topic() -> _notify() -> 防抖 200ms -> refresh_file_list() -> 增量更新 UI
```

**可能的原因**：
1. `_extract_topic_from_summary()` 的格式匹配失败（只匹配 `# ` 或 `## ` 开头）
2. 或者 `update_topic()` 未被调用（topic 为 None）
3. 或者防抖机制导致刷新时机不对

**修复方案**：

文件：`C:\侧耳倾听\src\gui\transcription.py`

增强 `_extract_topic_from_summary()` 方法（第 623-635 行）：
```python
def _extract_topic_from_summary(self, summary):
    """从摘要中提取会议主题"""
    try:
        lines = summary.splitlines()
        for line in lines[:10]:
            line = line.strip()
            # 匹配 "# 标题" 或 "## 标题" 格式
            if line.startswith("# ") or line.startswith("## "):
                topic = line.lstrip("# ").strip()
                if len(topic) > 5:
                    return topic[:50]
            # 匹配 "主题：xxx" 或 "**主题**：xxx" 格式
            if "主题" in line:
                m = re.search(r'主题[：:]\s*(.+)', line)
                if m:
                    topic = m.group(1).strip()
                    if len(topic) > 2:
                        return topic[:50]
    except Exception:
        pass
    return None
```

## 问题 D：评分全部显示 1

**现象**：页面上所有来自文件的评分都显示 1。

**根因分析**：

voiceprint_page.py 中显示的是 `quality`（样本质量），不是 `score`（匹配置信度）。

**关键发现**：
- `quality` 存储在 `voiceprint_library.json` 的每个 embedding 中
- 默认值为 0.85（`DEFAULT_SPK_QUALITY`）
- 如果所有样本都是手动添加或默认值，quality 会相同

**可能的原因**：
1. 所有样本的 quality 都是默认值 0.85，显示时四舍五入为 1
2. 或者显示逻辑有问题（格式化代码错误）
3. 或者 `quality` 字段未正确存储/读取

**修复方案**：

文件：`C:\侧耳倾听\src\gui\voiceprint_page.py`

调试步骤：

在 `_show_speaker_detail()` 方法中添加日志（第 748-750 行）：
```python
if profile.embeddings:
    for idx, emb in enumerate(profile.embeddings):
        logger.debug(f"[VOICEPRINT] Sample {idx}: quality={emb.get('quality', 'N/A')}")
    avg_quality = sum(e.get('quality', DEFAULT_SPK_QUALITY) for e in profile.embeddings) / len(profile.embeddings)
    logger.debug(f"[VOICEPRINT] Average quality: {avg_quality}")
    info_items.append(("平均质量", f"{avg_quality:.2f}"))
```

检查 `voiceprint_library.json`：
1. 打开文件，检查每个说话人的 `embeddings` 数组

3. 如果都是 0.85，说明没有计算真实质量

如果 quality 全是默认值，需要在转写时计算真实质量。

## 问题 2：API Key 提示显示错误

**现象**：删除 API Key 后显示"未配置 API Key，AI 功能不可用"，而不是"使用内置 Key"。

**根因**：两个独立 bug
1. `config.py` 第 69 行 `self._data.update(saved)` 用空字符串覆盖默认值
2. `settings_page.py` 的 `_on_save()` 保存后从未调用 `setText()` 更新提示

**修复方案**：

文件 1：`C:\侧耳倾听\src\config.py`
```python
def _load(self):
    if os.path.exists(self._path):
        with open(self._path, "r", encoding="utf-8") as f:
            saved = json.load(f)
        # 过滤空字符串值，防止覆盖 DEFAULTS 中的有意义默认值
        filtered = {k: v for k, v in saved.items() if v != "" or k not in DEFAULTS or DEFAULTS[k] == ""}
        self._data.update(filtered)
```

文件 2：`C:\侧耳倾听\src\gui\settings_page.py`
```python
# 在 _on_save() 方法的 config.save() 之后添加：
has_user_key = bool(self._config.get("ai_user_api_key", ""))
has_default_key = bool(self._config.get("ai_default_api_key", ""))
if has_user_key:
    self._api_key_hint.setText("已配置自定义 Key")
    self._api_key_hint.setStyleSheet(f"color: {C_SUCCESS}; font-size: 11px;")
elif has_default_key:
    self._api_key_hint.setText("使用内置 Key（不可查看）")

else:
    self._api_key_hint.setText("未配置 API Key，AI 功能不可用")
    self._api_key_hint.setStyleSheet(f"color: {C_ERROR}; font-size: 11px;")
```
