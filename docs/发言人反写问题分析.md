## 3. 发言人反写的两个场景

### 场景 1：有音色识别 → 直接写入转写

```
转写完成 → 声纹匹配 (_match_voiceprints) → 写入转写文件 (_apply_voiceprint_match)
```

**当前状态**：✅ 正常工作

- 声纹匹配成功后，`_apply_voiceprint_match()` 直接将姓名写入转写文件
- 同时写入 `_voiceprint_match_results` 供后续使用

### 场景 2：无音色识别 → AI 总结后反写

```
转写完成 → AI 摘要生成 → 从摘要提取姓名 → 写入转写文件
```

**当前状态**：❌ 有两个问题

#### 问题 2.1：时序错误

摘要生成在声纹匹配之前执行：

```
Worker: 写文件 → file_done → auto_summary → done
                               ↓
主进程:              生成摘要（此时 Speaker N）→ 声纹匹配 → 姓名应用
```

- `_generate_summary()` 在 `auto_summary` 消息处理时调用
- 此时 `_voiceprint_match_results` 为空
- AI 读取的转写文件还是 `Speaker N`

#### 问题 2.2：AI 输出格式错误

AI 输出 `[姓名]` 而非 `[Speaker N]`：

```
实际输出: [韩镭]（角色推断：项目负责人）
期望输出: [Speaker 1] 韩镭
```

导致 `_extract_speaker_mapping_from_summary()` 的 regex `\[Speaker\s+(\d+)\]` **完全匹配不上**。

#### 问题 2.3：姓名提取后覆盖声纹匹配

`_generate_summary()` 中的姓名提取逻辑（line 644-655）：

```python
speaker_mapping = self._extract_speaker_mapping_from_summary(summary)
if speaker_mapping:
    apply_speaker_mapping(item.result_path, speaker_mapping)  # ← 会覆盖声纹匹配的姓名
```

**没有检查**该 Speaker 是否已被声纹 confirmed 匹配。

### 改动 4：防御性修复摘要姓名格式

**文件**：`transcription.py:687+`

`_extract_speaker_mapping_from_summary()` 添加括号过滤和重复姓名去重：

```python
# 匹配 [Speaker N] 姓名 格式（支持 — 角色描述 后缀）
m = re.match(r'\[Speaker\s+(\d+)\]\s+(.+?)(?:\s*[—\-]\s*.*)?$', content)
if m:
    spk_num = int(m.group(1))
    name = m.group(2).strip().strip('*_#,，。')
    # 过滤角色推断：以括号开头的不是姓名
    if name and name.startswith(("（", "(")):
        continue
    # 过滤重复姓名（如 "嘉诚 嘉诚" → "嘉诚"）
    if name and " " in name:
        parts = name.split()
        if len(parts) == 2 and parts[0] == parts[1]:
            name = parts[0]
    if name and len(name) < 50:
        mapping[spk_num] = name
        continue
```
