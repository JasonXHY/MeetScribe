## 二、需要讨论的问题

### 2.1 设置页 AI 增强 section 功能范围

**现状**：新版 AI section 仅有 API Key + 摘要开关 + 纠错开关

**原版有**：
1. 模型厂商选择（ComboBox，联动模型列表）
2. 摘要模型选择（ComboBox，根据厂商变化）
3. API Key（LineEdit + 可见性切换按钮）
4. 接入模式选择（Token Plan / 按量计费）
5. 本地 LLM 选择（关闭 / Ollama）
6. 自动摘要模式（关闭 / 自动生成 / 手动触发）
7. 转写纠错模式（关闭 / 自动纠错）

**问题**：这些功能是否需要在本次迁移中全部实现？

**建议**：如果用户主要使用 MiMo API，可以简化为只保留 API Key + 开关，厂商选择等功能后续再加。

### 2.2 AddVoiceDialog 录音功能

**现状**：新版仅使用 QInputDialog 获取姓名，创建空的 SpeakerProfile

**原版有**：完整的录音弹窗，包含：
- 姓名输入
- 预设朗读文本显示
- 录音按钮（开始/停止）
- 录音状态显示
- 声纹提取进度
- 保存按钮

**问题**：录音功能是否需要在本次迁移中实现？

**建议**：录音功能需要与 UnifiedRecorder 集成，复杂度较高。可以先用"从文件导入"替代"现场录音"，后续再添加录音功能。
