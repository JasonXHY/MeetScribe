# MeetScribe 用户手册

> 版本：v0.6（2026-06-02）
> MeetScribe 是一款基于 AI 的会议录音转写助手，支持语音识别、说话人分离、标点恢复、AI 摘要生成和音色库功能。

---

## 一、环境要求

- 操作系统：Windows 10/11（64 位）
- Python：3.12+
- 内存：建议 8GB 以上（模型推理需要约 3GB）
- 磁盘空间：模型缓存约 4GB
- FFmpeg：需要安装并加入系统 PATH（用于音频格式转换）

### 安装依赖

```bash
pip install funasr modelscope torch torchaudio customtkinter sounddevice soundfile openai
```

### 安装 FFmpeg

```bash
winget install Gyan.FFmpeg
```

安装后重启终端，确保 `ffmpeg` 命令可用。

---

## 二、启动应用

双击 `MeetScribe.bat` 或运行：

```bash
python C:\MeetScribe\src\main.py
```

首次启动时需要下载模型（约 4GB），请在设置页点击"检查模型"并下载缺失模型。

---

## 三、主界面

主界面分为三个区域：

### 顶部工具栏
- **录音控制**：开始/暂停/停止录音，显示录音时长
- **输出格式**：选择转写结果的输出格式（Markdown、HTML、纯文本、SRT 字幕、JSON）
- **AI 摘要**：打开已完成转写文件的 AI 摘要
- **合并转写**：选中多个文件后合并为一份转写结果

### 文件列表
显示已添加的音频文件，包含以下信息：
- 文件名
- 时长
- 大小
- 状态（待转写 / 处理中 / 已完成 / 失败）
- 操作按钮（预览 / 打开 / 发言人 / 转写）

### 底部日志区
实时显示操作日志和转写进度。

---

## 四、录音

1. 点击"开始录音"按钮
2. 录音过程中可以暂停和继续
3. 点击"停止录音"结束
4. 录音文件自动保存到录音目录
5. 停止后会提示是否立即转写

录音文件默认保存为 WAV 格式（16kHz 单声道）。

---

## 五、添加音频文件

### 方式一：添加文件按钮
点击工具栏"添加文件"，选择一个或多个音频文件。支持格式：WAV、MP3、M4A、FLAC、OGG。

### 方式二：录音结束后自动添加
录音停止后会自动添加到文件列表。

---

## 六、转写

### 单文件转写
在文件列表中点击"转写"按钮。

### 批量转写
选中多个文件（点击文件行选中），点击工具栏"开始转写"。

### 合并转写
选中多个文件，点击"合并转写"，可以调整文件顺序后合并为一份结果。

### 转写过程
转写在后台进程中运行，UI 保持响应。进度会在状态栏和日志区实时显示：
1. 加载模型
2. 文本识别 + 说话人分离 + 标点恢复
3. 转写完成
4.（可选）AI 摘要生成

---

## 七、查看结果

### 预览
点击文件行的"预览"按钮，弹出预览窗口：
- **转写结果**标签页：显示格式化的转写文本
- **AI 摘要**标签页：显示 AI 生成的会议摘要（需提前配置 MiMo API）

### 打开文件
点击"打开"按钮，用系统默认程序打开结果文件。

### 发言人管理
点击"发言人"按钮，可以：
- 查看识别出的说话人列表
- 为说话人分配真实姓名
- 批量替换说话人名称

---

## 八、设置

### 模型管理
- **检查模型**：检查所有必需模型是否已缓存
- **下载缺失模型**：自动下载未缓存的模型
- 模型状态显示：每个模型显示已缓存/未缓存状态

### 情感识别
- 开关：开启后转写时会额外进行情感分析
- 需要下载 emotion2vec 模型（约 1GB）
- 情感结果会显示在转写结果的元数据中

### AI 配置
- **MiMo API Key**：用于 AI 摘要生成
- **摘要模型**：选择 mimo-v2.5-pro（高质量）或 mimo-v2.5（性价比）
- **自动摘要**：开启后转写完成自动生成摘要

### 目录设置
- **录音目录**：录音文件保存位置
- **输出目录**：转写结果保存位置

## 九、输出格式

| 格式 | 说明 | 适用场景 |
|------|------|---------|
| Markdown | 按时间线合并连续发言，带说话人标记 | 人工阅读 |
| LLM Markdown | 每句独立标记，适合 AI 处理 | 送入 AI 做二次分析 |
| HTML | 带颜色标记的网页版 | 浏览器打开，分享 |
| 纯文本 | 简单文本格式 | 复制粘贴 |
| SRT 字幕 | 标准字幕格式 | 视频字幕 |
| JSON | 结构化数据 | 程序处理 |

## 十、音色库功能

音色库功能可以通过声纹比对自动识别已注册的说话人，让用户在多次会议中遇到同一人时无需每次手动标注。

### 10.1 注册说话人

1. 完成转写后，点击文件行的"发言人"按钮
2. 在弹窗中为每个 Speaker 输入姓名
3. 点击"保存到音色库"按钮
4. 系统会自动提取声纹并保存

### 10.2 自动匹配

- 转写完成后，系统会自动与音色库中的声纹进行比对
- 匹配成功会显示"（音色库）"标记
- 本地音色库优先级高于 AI 识别

### 10.3 管理音色库

在设置页可以查看和删除已注册的说话人。

### 10.4 准确性提升

- 每次会议都会增加声纹样本
- 样本越多，匹配越准确
- 建议至少 3 个样本后才激活匹配

## 十一、常见问题

### Q: 转写时 UI 卡死 / 未响应
A: 当前版本使用多进程架构，UI 不应卡死。如果出现此情况，请检查日志文件 `C:\MeetScribe\logs\meetscribe.log`。

### Q: 模型下载失败
A: 检查网络连接。模型从 ModelScope 下载，需要稳定的网络。可以尝试在设置页重新点击"下载缺失模型"。

### Q: 说话人识别不准确
A: 说话人分离依赖音频质量。建议：
- 使用清晰的录音设备
- 避免多人同时说话
- 会议环境尽量安静

### Q: 标点恢复没有效果
A: 标点恢复由 ct-punc 模型处理。如果转写结果没有标点，可能是模型加载失败。检查日志中的错误信息。

### Q: 情感识别内存不足
A: emotion2vec 模型已移除，不再占用内存。

### Q: 音色库匹配不准确
A: 音色库匹配依赖声纹质量。建议：
- 至少注册 3 个样本后再激活匹配
- 保持录音环境安静
- 每次会议都注册说话人，提高准确性

### Q: 如何停止转写
A: 在转写过程中，点击文件行的"停止"按钮即可停止当前转写任务。

### Q: FFmpeg 未找到
A: 请安装 FFmpeg 并确保其在系统 PATH 中：
```bash
winget install Gyan.FFmpeg
```
安装后重启应用。
